# autoFDX（Fallen Doll 自动化）

> 当前版本已支持模块化架构、悬浮控制台、逐项标定、自适应分辨率/比例、进度条纠偏、可选点赞策略与全屏兼容实验开关。

---

## 1. 项目简介

`autoFDX` 是一个基于图像识别与鼠标自动化的辅助脚本，核心目标是：

- 在不同分辨率、缩放、显示模式下尽量稳定运行；
- 通过用户标定提升模板匹配精度；
- 用悬浮窗提供可视化控制、状态显示与热键安全暂停；
- 在流程关键阶段做进度条平衡纠偏，提升触发稳定性。

---

## 2. 环境与安装

### 2.1 Python 版本

- 推荐：`Python 3.10+`（你当前项目实际在 Windows 环境下运行）

### 2.2 依赖安装

推荐小白用户直接双击：

- `setup_windows.bat`

如需手动安装，在项目根目录执行：

```bash
pip install -r tools/requirements.txt
```

> 说明：`threading`、`sys`、`winsound` 为 Python 标准库，无需 `pip install`。

## 4. 悬浮控制台说明

### 4.1 标题区

- `autoFDX 控制台`
- `按“F1”键紧急暂停`（提示文案）

### 4.2 右上角按钮

- `全屏模式兼容`（勾选）
  - 开启后启用实验性 Overlay 兼容路径（Win32 扩展样式）
  - 用于提升全屏/无边框全屏下的悬浮可见性
- `悬浮中 / 悬浮`（按钮）
  - 控制是否保持悬浮显示（topmost）
  - `悬浮中` 时禁止拖动窗口
- `✕`
  - 退出脚本

### 4.3 主控制按钮（大按钮）

- `继续 / 暂停`
  - 显示 `继续`：当前处于暂停状态（按钮为绿色）
  - 显示 `暂停`：当前处于运行状态（按钮为红色）
- `自定义标定`
  - 展开/收起标定菜单
- `退出脚本`
  - 停止并退出程序

### 4.4 点赞相关勾选

- `启用点赞功能`
  - 关闭后不会调用点赞流程
- `结束后执行点赞`
  - 一次性触发：下一次流程点击结束后立即执行点赞
  - 执行后自动清除该标记

## 5. 标定按钮含义（逐项说明）

展开 `自定义标定` 后可见（红=未标定，绿=已标定）：

- `开始按钮`
  - 标定“开始”模板图与匹配区域
- `高潮按钮`
  - 标定“高潮”模板图与匹配区域
- `再来一次按钮`
  - 标定“结束/再来一次”模板图与匹配区域
- `女进度条`
  - 对应原上方进度条（bar1），保存区域 + 颜色采样
- `男进度条`
  - 对应原下方进度条（bar2），保存区域 + 颜色采样
- `调速区域`
  - 滚轮纠偏前，鼠标先移动到该区域中心
- `用户1 / 用户2 / 用户3`
  - 点赞流程中的用户入口点位
- `点赞用户1 / 点赞用户2 / 点赞用户3`
  - 点赞动作对应点位

### 5.1 标定操作方式

- 普通项（按钮/进度条/调速区）：矩形框拖动/缩放
- 点赞项：圆点模式（只保存点击点）
- 快捷键：
  - `Enter` 保存当前标定
  - `Esc` 取消当前标定

## 6. 自动流程概览

主流程大致为：

1. 等待并点击 `开始`
2. 进入“开始后到高潮前”阶段
   - 主线程检测状态与按钮
   - 副线程按主线程下发参数执行滚轮纠偏
3. 识别到 `高潮` 后停止纠偏并快速点击
4. 等待并点击 `结束`
5. 按规则触发点赞（见下一节）

---

## 7. 点赞触发规则

当前逻辑：

- 默认每 **5** 次主流程触发一次点赞；
- 若勾选 `结束后执行点赞`，则在下一次流程结束后立即点赞一次；
- 该次立即点赞执行后，点赞计数归零，后续继续按 5 回合周期触发。

## 8. 性能与稳定性说明

已做的关键优化：

- 模板图缓存（避免重复磁盘读取）
- 标定区域优先局部截图匹配（避免频繁全屏截图）
- 进度条独立标定（女/男）降低误判
- 滚轮与主检测线程职责分离（关键点击优先）
- UI 位置持久化、展开收起高度自适应

## 9. 常见问题

### 9.1 为什么全屏看不到悬浮窗？

- 真独占全屏下普通窗口很难叠在最上层；
- 建议优先使用无边框全屏；
- 可尝试勾选 `全屏模式兼容`（实验）。

### 9.2 程序异常退出怎么排查？

- 查看：`logs/error.log`

### 9.3 点击“继续”表现异常？

- 目前按钮已加防抖，若仍异常可看日志并反馈触发路径。

---

## 10. 使用建议

- 首次使用先完整标定一遍关键项（开始/高潮/结束/女条/男条/调速区）；
- 游戏画面设置变动较大（亮度、HDR、UI比例）后建议重新标定；
- 遇到误识别优先缩小标定区域，减少背景干扰。
