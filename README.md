# autoFDX（Fallen Doll 自动化）

> 模块化架构、悬浮控制台、逐项标定、自适应分辨率/比例、进度条纠偏、可选点赞与赞池检测、**实验切换模式**、女进度条停滞恢复、敏感度不足自动补偿、「拉出新实验滚动」可配置与 **F11 重播**、**5 秒平滑移动视角部署**（匀速右移 + 连续左键，整圈视角标定）、**F2 恢复后切换实验时按「恢复体力按钮」判图循环 ESC**、可选 **自动补充体力**（体力不足图标 + 独立补充按钮 + 凝胶确认）等。

---

## 1. 项目简介

`autoFDX` 是一个基于图像识别与鼠标自动化的辅助脚本，核心目标是：

- 在不同分辨率、缩放、显示模式下尽量稳定运行；
- 通过用户标定提升模板匹配精度；
- 用悬浮窗提供可视化控制、状态显示与热键安全暂停；
- 在流程关键阶段做进度条平衡纠偏，提升触发稳定性；
- 支持实验切换模式，自动顺序完成多个实验并按需切换；
- 实验部署阶段支持 **SendInput** 底层输入与 **整段时长内平滑视角扫圈**，便于对准可部署区域。

## 2. 环境与安装

### 2.1 Python 版本

- **推荐：`Python 3.12`**（与 `tools/setup_windows.ps1` 中安装/检测版本一致）。
- 最低建议：`Python 3.10+`（Windows）。

### 2.2 依赖说明

依赖**显式**列在 **`tools/requirements.txt`**（唯一完整清单），包括：

| 类别 | 包名（节选） |
|------|----------------|
| 视觉 / 数值 | `numpy`、`opencv-python` |
| 自动化 / 截图链 | `Pillow`、`pyscreeze`、`pymsgbox`、`pytweening`、`mouseinfo`、`pyperclip`、`pyrect`、`pygetwindow`、`pyautogui` |
| 热键 | `keyboard` |

根目录 **`requirements.txt`** 仅一行 `-r tools/requirements.txt`，便于 `pip install -r requirements.txt` 与上述清单保持同步。

### 2.3 安装方式

**推荐（小白）**：双击项目根目录 **`setup_windows.bat`**，将调用 `tools/setup_windows.ps1` 完成：

- 检测或安装 Python 3.12；
- `pip install -r requirements.txt`（或回退为 `tools/requirements.txt`）；
- 校验关键模块可导入。

**手动安装**（在项目根目录）：

```bash
pip install -r requirements.txt
```

或直接：

```bash
pip install -r tools/requirements.txt
```

> `threading`、`json`、`tkinter`、`ctypes`、`pathlib` 等为 Python 标准库或系统组件，无需 `pip install`。

## 4. 功能要点（与文档对应）

| 能力 | 说明 |
|------|------|
| 实验切换 | 选定实验 → 部署 → 正常运行 → 满 5 回合分支（切换下一实验 / 当页完成拉新行） |
| **移动视角部署** | 开始按钮未出现时，在约 **5 秒**内 **匀速向右平滑移动视角**（累积位移对标约 **360°**，结合屏宽与系统 DPI），并 **连续左键** 尝试部署；同时轮询「开始 + 恢复体力」模板 |
| **F2 → F1** | `F2` 暂停并置位「恢复后切换下一实验」；`F1` 恢复后优先执行切换。首次 `ESC` 后若 **恢复体力按钮** 仍可见，则短暂延迟后 **循环 `ESC`** 直至该按钮消失，再进入 `ESC → 延时 → E → …` 的常规切换序列 |
| 自动补充体力 | 控制台勾选且完成「体力不足图标」「体力补充按钮（独立）」「使用凝胶确认」等标定后，在点击「开始」前按需补体力 |
| 女条停滞 / 敏感度补偿 | 独立后台逻辑，见 `详细流程逻辑.md` |
